# Cross-pericope fluency probe — findings

**Date:** 2026-07-19
**Question:** The content bank is built on the smallest unit, the pericope. How do we
work the fluency of *timeline, events, and logic progressions that cross pericopes*
within a book — and does that need any change before the full content-bank build starts?
**Method:** Empirical probe, same play as the genre probe
(`docs/2026-07-19-genre-dimension-probe-findings.md`). Hand-author a small set of
**cross-pericope** items (each unanswerable from any single pericope) across all eight
dimensions D1–D8 for one narrative span and one epistle span, using real BSB
public-domain text served through the corpus license gate under the WCF-1 guardrail.
Observe which dimension houses each item and whether a load-bearing cross-pericope skill
is *orphaned* (fits no dimension), then resolve four decisions (D-A..D-D) into a go/no-go
for scaling the bank.

**Spans.** Narrative: **Matthew's infancy arc**, the eight already-seeded pericopes
MAT-001..008 (genealogy → birth → Magi → flight to Egypt → weeping in Ramah → return to
Nazareth → John the Baptist → baptism). Epistle: **the whole of Philippians**, divided
into seven *probe-local, provisional* pericopes (NOT committed to the corpus), extending
the genre probe's Philippians 2:1–11 to the letter's full argument.

## Starting thesis (H1, pressure-tested)

Cross-pericope fluency needs **no new dimension and no new stored unit**: it is D2
stretched to book-timeline, D5 stretched to arc-connection, and D7 stretched to
book-argument, with the selector composing the book-level moves at kit-build time from
corpus ordering + cross-refs + already-authored objectives. Under H1 the full build
proceeds immediately with no per-pericope change. **The probe wins by *breaking* H1**
(finding an orphan or a required per-pericope hook), not by confirming it.

| **D8 Application** | As a family, draw the journey (Bethlehem → Egypt → Nazareth) on one card and place each fulfilment quotation beside the event it lands on; each person names one aloud. | **✓ load-bearing** — a concrete, observable, doable response; the arc content is presupposed, not taught, by D8. |

**Tally: 5 ✓ · 2 △ · 1 ✗.**

**Orphan finding.** A load-bearing cross-pericope narrative skill is orphaned: the
**book-timeline / event-chain / "where are we in the story" skill.** The single dimension
whose content *is* order-and-causation, **D2, refuses it by its own rule** — the D2 item
is authorable only by violating "Avoid sequence spanning other pericopes." Nothing else
picks it up: D5 houses *named* inter-text echoes, not a who-fled-where-then-what chain;
D7 houses book-scale *interpretation*, the *why*, not the *timeline of what happened*; D4
houses *recall of a set*, which degrades to D2 the moment "in order" is asked. The ✓
dimensions all house the *products* of arc comprehension (recalled phrases, named echoes,
interpretive payoff, applied activity) while **none houses the comprehension itself** —
each ✓ item presupposes the learner already holds the story's spine. Just as metaphor fell
through in Psalm 23, the bare narrative spine falls through here.

## Result — Philippians (epistle argument)

The *phroneo* / "mind" thread is one Greek verb (φρονέω) that BSB renders with a different
English surface each time (1:7 "feel this way" · 2:2 "like-minded" · 2:5 "this mind" ·
3:15 "point of view"/"think" · 4:2 "agree") — invisible to English phrase-matching. Joy
bookends verified at 1:18 and 4:4. Verdicts post-review.

| Dim | Representative cross-pericope item | Verdict |
|---|---|---|
| **D1 People & Places** | Assemble the letter's named cast and each role: Paul & Timothy (1:1), Timothy to be sent (2:19–23), Epaphroditus (2:25–30), Euodia & Syntyche (4:2–3), Clement, the household of Caesar (4:22). Who appears where? | **△ forced** — only the *union* of per-pericope rosters; no genuinely pericope-crossing inference, so the "arc" framing is cosmetic. |
| **D2 Event Sequence** | Trace the argument chain 1:27 "conduct worthy… stand firm" → 2:1–4 "be of one mind… in humility" → 2:5–11 "Let this mind be in you, which was in Christ." What does each step license the next to assert? | **✗ no home** — this is *logical-dependency* tracing, not the **event** order D2 covers (there are no events), and D2's anti-span rule bars it regardless. Fails on both counts. |
| **D3 Vocabulary** | The *phroneo* thread (1:7, 2:2, 2:5, 3:15, 4:2): one key term carrying the letter's plea for a shared mind. What does the repeated word *do* across the chapters? | **△ forced** — the load is the term's *argumentative function across pericopes* (and BSB varies the surface, so it isn't even a matchable phrase); that is D7-argument work in a vocabulary hat. |
| **D4 Memory** | Recall the rejoicing refrain and cite where it recurs: 1:18; 3:1; 4:4 ("I will say it again: Rejoice!"). | **✓ load-bearing** — recalling a recurring memorable phrase is native D4; a memory set can span locations. |
| **D5 Connections** | Name the text behind 2:10–11's "every knee… every tongue" — **Isaiah 45:23**; and note the internal echo 1:27 "stand firm" ↔ 4:1 "stand firm." | **△ forced** *(corrected on review from ✓)* — the ✓-worthy Isaiah link is **single-pericope** (2:10–11 both in PHP-p4), so it is not cross-pericope; the genuinely cross-pericope element (the 1:27↔4:1 echo) is D5 **borrowing the cross-ref hat for argument-thread tracing**. |
| **D6 Questions** | Paul rejoices in chains, possibly facing death (1:18–23), yet commands "Rejoice always" (4:4). | **D7 Interpretation** | Where does the 2:5–11 hymn sit in Paul's argument? "Let this mind be in you" (2:5) grounds the 2:1–4 humility call, which serves the 1:27 exhortation; the humiliation→exaltation the verses state is the warrant. | **△ forced** — D7's top rung reaches book-argument, so it is the nearest catchment, but its *unit is one passage*; made to carry the multi-seam trace as the object of study it houses the endpoint by **overload**, not natively. |
| **D8 Application** | Name one concrete interest of another family member you will put ahead of your own (2:3–4), and rejoice aloud about one hard circumstance (1:18; 4:4). **Tally: 3 ✓ · 4 △ · 1 ✗.** WCF-1 note: the D7 item stays strictly on what 2:6–11
*states* ("emptied Himself," "obedient to death," "God exalted Him," "every tongue confess
that Jesus Christ is Lord") and does not adjudicate any kenosis theory the verses leave
open.

**Orphan finding.** Cross-chapter **argument / thread-tracing** is orphaned: D2 refuses it
twice (no events to order + the anti-span rule); D3 catches only the surface term
(stretched); D7 is the nearest home and takes the multi-seam trace by **overload**,
housing the endpoint ("what the hymn means for unity"), not the connective path. This is
the **same** orphan the genre probe already found — intra-pericope argument-connective
absorbed into D3/D7 — now at **book scale and strictly worse**, because the cross-pericope
span additionally trips D2's anti-span rule the intra-pericope version never touched.

## Consolidation — the two orphans are one

Both spans locate the same underlying gap: **holding the connective tissue across pericope
seams** — temporal/causal in narrative, logical (grounds→therefore) in an epistle. In both,
(a) D2 is the natural owner and affirmatively refuses via the identical one-line rule; (b)
the ✓ dimensions house the arc's *products* while none houses the comprehension; (c) it is
the genre probe's argument-connective orphan at book scale, made worse by D2's anti-span
rule. One consolidated orphan: **cross-pericope structural comprehension.** The refusal is
real *today* — under the current authoring contract there is literally no dimension an
author may file a cross-pericope sequence item under. But it is a one-line *drafting-
guidance* rule, not an ontological claim that the skill cannot be scored — which is what
makes the fix a reshape/schedule move rather than a new dimension.

## Decisions

**D-A — does authoring a pericope need a new hook now (the only build blocker)? → NO.**
Tested against every reviewed cross-pericope item: each is authorable from material that
already exists, with nothing stored on the pericope at authoring time.
- The **Matthew D2 orphan item** ("order the infancy events; Herod's decree → flight") is
  reconstructable entirely from `mat.json`'s ordered array + each pericope's verse `range`
  + the served BSB text (the Herod→flight causation is *stated* in 2:13/2:16), with zero
  reference to any stored arc-role.
- The **least corpus-derivable item — the Philippians *phroneo* thread** (its one-verb-
  many-surfaces insight is real external lexical knowledge, absent from ordering, ranges,
  served English, and `crossrefs.json`) — still needs no per-pericope field: it belongs in
  a later **leader-reference / arc artifact** that references stable pericope ids, not in
  each ContentItem. Nothing about authoring PHP-p1..p7 individually must change to let that
  thread be traced later.
- **Schema check:** `schema.py`'s ContentItem has no arc-role/section field and `passage`
  resolves to a *single* pericope; the finding is that we **don't need** to add one. The
  orphan is a **scheduling / authoring-move gap**, not a per-pericope-data gap.

**D-B — where cross-pericope content lives. → A derived spine + a separate arc artifact;
NOT span-scoped ContentItems.** Mechanical moves ("story so far," "put in order,"
roster-union, "where we are in the argument") are **derived-only, composed by the selector
at kit-build** — sanity-checked against `selector.py`, which already receives the ordered
`bank["pericopes"]` and `family["reading_sequence"]` and already composes review across all
studied passages; emitting a recap is the same shape of move (it would additionally read
`crossrefs.json`, a small additive read). Interpretive arc content (what the refrain *does*,
the *phroneo* function, the hymn's warrant) carries authored judgment and belongs in a
**lamppost-like arc artifact** (the corpus already has the lamppost precedent). Span-scoped
`passage` is rejected: it would break the one-pericope-per-item invariant or force the D-D
section entity we otherwise avoid.

**D-C — session shape (a finding). → Both.** A **recurring one-line recap micro-segment**
on every pericope kit ("The story so far…", "Where this sits in Paul's argument…") keeps
the spine warm and is cheaply derivable; the **heavy items** (order all eight infancy
events, draw the journey card, trace the whole *phroneo* thread) are too large for a
micro-segment and read as a **periodic dedicated "zoom-out" review session** at a
section/book seam — where the orphan's comprehension is actually exercised and observed.
Both are Prepare-authored onto paper; neither is a during-session computing feature
(`unplug_assitant.md` §16).

**D-D — does the corpus need a mid-level section grouping? → No stored layer now; not
blocking.** `structure/` confirms the wall (book → flat ordered array of pericopes,
nothing between). Contiguous windows (last N pericopes, a bounded range) are derivable from
ordering + ranges — enough for the recap and a rolling zoom-out. Only the *boundaries* of a
theologically meaningful movement (does the infancy end at MAT-006, or does John the
Baptist open a new section?) are editorial judgment ordering can't supply — and those can be
authored **in the arc artifact** by referencing ids. A stored section layer becomes a
corpus change only if such grouping must be queryable structure shared across selector and
arc layer, and even then it is additive to `structure/`, not a per-pericope field — so it
still never gates pericope authoring.

## Implications for design — GREEN LIGHT

**The full pericope build and the arc layer proceed independently; the arc layer is a
later, additive spec.** The only two things that could have gated the mass build — a
per-pericope hook (D-A) and a stored corpus section layer (D-D) — both resolve as
not-required. Every reviewed cross-pericope item, including the orphan itself, is authorable
from corpus ordering + `crossrefs.json` + existing objectives/text. **Scaling pericope
authoring now bakes in no cross-pericope gap that a retrofit would have to undo,** because
the missing piece is a compose/schedule/present layer that reads already-existing data, not
anything stored per pericope.

**The orphan's home is a book-arc SCHEDULING / RESHAPE move over D2/D5/D7 — not a new
dimension (D9).** Consistent with the genre-probe precedent (reshape/reweight/schedule
before adding), and for concrete reasons:
- The orphan's *products* already have homes (D4 recalled set, D5 named echoes, D7 book-
  argument, D8 arc activity); what is missing is a **composition move** that assembles them
  across seams and a **surface** (recap + zoom-out) that shows the connective spine.
- A **D9 would re-introduce the very per-pericope hook D-A rejects** (its own per-item
  authoring and assessment ladder) and fits the per-pericope ContentItem model poorly,
  because the skill is inherently cross-pericope / cross-session.
- D2 "refuses" only via a **one-line template rule**. Lift it *only* in an explicitly
  arc-scoped context and D2 houses book-timeline natively.

The cheapest correct future treatment (all additive, all later, none blocking the build):
1. a **context-scoped reshape** of the D2 template so arc scope permits cross-pericope
   sequence, leaving per-pericope authoring's "avoid cross-pericope sequence" rule intact;
2. **selector scheduling** of the recap micro-segment + periodic zoom-out session;
3. the **lamppost-like arc artifact** for interpretive thread content, carrying any named
   section boundaries.

**Honest counter-weight.** The probe shows D2/D5/D7 stretched house only the *products*,
not the comprehension — the strongest argument *for* a D9. Judged insufficient: this is a
**representation + scheduling** gap (no surface shows the spine, no move composes it), not a
**taxonomy** gap (no missing kind of observable skill). If, once the arc layer is built and
used, the longitudinal record still cannot distinguish "reconstructs the plot" from
"memorized five quotations," *that* is the empirical trigger to revisit D9 — future
evidence, not a reason to block or complicate the build now.

**Watch-items for the arc-layer spec (none blocking):** keep the D2 "avoid cross-pericope
sequence" rule as-is for the mass build (it is the one string the arc layer will reshape,
contextually); D-B's interpretive residue and D-D's named boundaries are real authored work
the arc layer must fund (the green light covers the *pericope* build, not a claim the arc
layer is free); D4's Matthew ✓ is conditional (checklist, not "in order") — route ordering
through the arc-scoped D2, don't over-read D4 as arc coverage.
